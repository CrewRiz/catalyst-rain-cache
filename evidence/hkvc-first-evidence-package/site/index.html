CATALYST HKVC: A Mixed-Fidelity, State-in-Payload Cache

Standard transformer KV-caches grow linearly (O(N)) with context length, and compressed alternatives introduce severe recency bias. CATALYST HKVC introduces a two-lane hypervector cache runtime to eliminate both bottlenecks.

  1. Exact Lane (Zero-Tax Decode): A payload-native exact state machine. When full semantic memory isn't required, decode kernels completely bypass accumulator allocation, tracking, and serialization. O(1) latency per token.
  2. Compact Semantic Lane (Recency-Unbiased): When long-context semantic compression is needed, exact state is explicitly promoted into SDK-owned fixed-size archives. The public repo reports behavior and boundaries; the private representation lives inside Catalyst Brain.
  3. State-in-Payload: Both lanes serialize entirely into RAIN (.rain) state packages, carrying computation state across ephemeral nodes without external database reads.

Saturation Boundary

A single accumulator is not advertised as a perfect 128K-token store. Crosstalk noise scales as sqrt(N / D), so the safe 3 sigma boundary is N ~= D / 9. For D = 4096, CATALYST HKVC chunks the compact lane at 455 tokens per accumulator and stores many bounded accumulators instead of one saturated field.

Drop-in Cache Contract

The Hugging Face cache bridge preserves standard tensor shapes. Exact mode returns concatenated tensors shaped (batch, heads, seq_len, head_dim). Compact semantic mode calls the Catalyst Brain SDK to materialize a bounded response shaped for standard attention layers. Causal masking is enforced in exact mode; compact state is treated as already-past accumulated knowledge.

Breakthrough Verdict

Fixed-State Quality Evidence

This is the primary O(1) evidence lane: fixed Catalyst state carries far-context recall while the active exact window remains bounded. The TinyLlama result is an effective-context integration: Catalyst recalls the far fact and materializes it into the active prompt; it is not a raw RoPE extension claim.

RULER/Needle-Style Suite

This synthetic suite exercises fixed-state far-context retrieval across single-needle, multi-needle, duplicate/latest-position, and wrong-position rejection tasks. The sliding-window baseline includes a recent-window positive control so its far-context miss rate is not mistaken for a broken baseline. Linear scan can retrieve retained records, but its memory and work grow with retained items.

Manifold Attention Gap-Closing Tier

This tier tests the SDK-owned deep-context attention path against a compact proxy baseline. The public repo publishes benchmark inputs, outputs, charts, and boundaries while keeping the implementation inside Catalyst Brain.

Lossless Replacement Tier

This tier separates universal exactness from compact semantic compression. The SDK retains exact state outside the bounded local tensor window and evaluates attention in bounded tiles, then compares the output against PyTorch scaled-dot-product attention.

Scaled Lossless Profiles

This tier broadens the exact-state path across decode overflow, causal prefill, additive masks, and a three-layer decode stack. It is operator-level evidence, not a full model-generation benchmark.

Tier-Two Evidence Package

This tier adds broader fixed-corpus tasks, model-level target perplexity, generation rubric scoring, and modeled comparisons against KIVI, TurboQuant, PyramidKV/H2O-style retention, vLLM/PagedAttention, and sparse or retrieval-based attention. It is not an official LongBench/RULER submission; it is a checked-in harness for the next evidence tier.

Next Evidence Tier

This tier tracks launch-grade evidence readiness: official LongBench/RULER source reachability, same-machine baseline import status, public adapter latency, and Cloudflare Workers AI stronger-model probing.

Official Benchmark Subset

This tier runs official-source LongBench v2 and RULER subsets through Cloudflare Workers AI. The checked RULER run uses an explicitly labeled answer-only prompt wrapper with the official answer prefix; this fixed the variable-tracking answer-budget failure while keeping the subset boundary visible. It is not a full official submission and is not a Catalyst adapter quality number until the runner is connected to the live Catalyst model path.

RAIN Worker Transport

The next serving path avoids raw-prompt payload failures by sending packed tokens or RAIN (.rain) state to a Worker that privately calls Catalyst Brain. The public adapter owns only the transport envelope and metadata; lossless binding, retrieval, rehydration, and active-context materialization remain behind the SDK boundary.

Optional Chunk Archive Evidence

This benchmark compares ordinary transformer KV residency against chunked HKVC archive memory for 128K, 512K, and 1M tokens. It is a saturation-safe semantic archive model, not the hot decode path and not the headline fixed-state claim.

Claim boundary: the fixed-state lane is the O(1) headline. Chunked archives are optional semantic storage with deterministic fixed-size chunks. Production RAIN (.rain) payloads should be encrypted and authenticated before client-held or cross-node transport.
